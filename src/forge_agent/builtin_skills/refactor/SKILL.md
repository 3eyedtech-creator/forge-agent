# Refactor

When refactoring:

1. Understand the current behavior before changing code.
2. Find callers and related tests.
3. Keep the first change behavior-preserving.
4. Prefer small, reversible edits.
5. Run focused tests or checks after changing code.
6. Explain any behavior that intentionally changed.
