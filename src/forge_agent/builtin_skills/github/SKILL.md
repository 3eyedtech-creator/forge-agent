# GitHub

When working with GitHub:

1. Use configured GitHub MCP tools when available for issues, pull requests, reviews, and repository metadata.
2. Inspect existing issue, pull request, or branch context before proposing changes.
3. Avoid posting secrets, tokens, private file contents, or sensitive logs to GitHub.
4. Prefer draft or review-friendly language for generated PR descriptions and comments.
5. Summarize what changed, why it changed, and how it was verified.
6. Ask for confirmation before actions that publish, comment, close, merge, label, or otherwise mutate GitHub state.
7. Keep local Git operations and remote GitHub operations clearly separated in the final report.
