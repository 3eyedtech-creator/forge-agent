# Bugfix

When fixing a bug:

1. Reproduce or understand the failure first.
2. Read the traceback, failing test, or error output carefully.
3. Retrieve and inspect the smallest relevant code area.
4. Form a specific hypothesis before editing.
5. Make the smallest safe change.
6. Run the most relevant verification command.
7. Report what changed, what passed, and any remaining risk.
